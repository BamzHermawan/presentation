Created by BamzHermawan

ABOUT PIA

Project Information Accuration (PIA) merupakan salah satu Project Management Information System(PMIS) yang bertujuan sebagai fasilitas (tools) untuk menunjang dalam pengelolaan project-project menjadi lebih baik dan terukur

DIMANA SERVER PIA ?

Saat ini server PIA berada di nutanix yaitu private cloud dengan menggunakan 2 server terdiri dari server apps dengan ram sebesar 8 gb & server database dengan 16 gb.

PIA FLOW SISTEM

ROLE USER

Pada flow proses sistem PIA di atas terdapat 6 role usernya yaitu :

    1. Project Manager Head (PMD)

    2. VP Delivery

    3. Head OF Delivery (HOD)

    4. Project Owner

    5. Project Manager

    6. Team Member

PROJECT MANAGER HEAD

(PMD ASSIGNMENT)

PMD Dapat Melakukan sbb :
  1. Melihat Workplan View
    2.
  2. Melihat Team Member Summary
    3.
  3. Melakukan Upload project Document
    4.
  4. Melihat S-curve
    5.
  5. Membuat Project Notes
    6.
  6. Set Unit Delivery & Effort Type (update)
    7.
  7. Assign Project Manager (update)
    8.
  8. Assign Co Project Manager (update)
    9.
  9. Assign Project Admin (update)
    10.

PMD menunjuk seseorang untuk di berikan akses profile PM Head dengan catatan pada unit yang sudah mendapatkan assignment project dan sudah di setujui oleh VP terkait.

ketika assignments telah masuk ke dalam list need assignment maka mereka dapat melakukan set unit, unit delivery & Effort Type, Assign PM, Assign Co PM, Assign Project Admin serta update.

HEAD OF DELIVERY

(HOD RESOURCE)

Pada Aktor ini dapat mengassign member kedalam project dan menentukan berapa banyak resource yang di pekerjakan pada project tesebut.

pada profile ini HOD dapat melakukan :

  1. Melihat Workplan View
    2.
  2. Melihat Team Member Summary
    3.
  3. Melakukan Upload project Document
    4.
  4. Melihat S-curve
    5.
  5. Membuat Project Notes
    6.
  6. Assign Resource Team Member ke Project
    7.

VP DELIVERY

(APPROVAL)

Aktor selanjutnya yaitu VP delivery

Beberapa aktifitas yang bisa dilakukan VP delivery di PIA sebenarnya hanya melakukan monitoring project seperti approval/reject jika ada req dan perubahan workplan yang di lakukan PM

secara keseluruhan VP Delivey dapat melakukan sbb :

  1. Dapat melihat dan memonitor Project List
    2.
  2. Approval For Start (Approve / Reject)
    3.
  3. Approval For Rebaseline (Approve / Reject)
    4.
  4. Approval For Closing (Approve / Reject)
    5.

KEGUNAAN SISTEM

PIA saat ini digunakan untuk pengukuran project - project baik di sisi penggunanya maupun dari management dimana sistem PIA di harapkan mampu memberikan efisiensi serta efektifitas

Dimana pada sistem sebelumnya masih mempunyai banyak kekurangan (Prouds) sehingga PIA di lahirkan sebagai cikal bakal pengganti prouds agar lebih akurat dan dapat menunjang performance kerja jauh lebih baik dan tentu PIA digunakan untuk project yang sudah ada IWO nya.
